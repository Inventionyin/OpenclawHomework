# 云服务器接手说明

## 1. 服务器用途

这台服务器用于部署 OpenClawHomework 的飞书 Webhook 桥梁服务。

当前链路：

```text
飞书 / 模拟 Webhook
  -> https://openclaw.evanshine.me/webhook/feishu
  -> Nginx
  -> Node 桥梁服务 127.0.0.1:8788
  -> 可选：OpenClaw 解析自然语言指令
  -> GitHub API workflow_dispatch
  -> GitHub Actions 执行 UItest
```

## 5. 服务信息

systemd 服务名：

```text
openclaw-feishu-bridge
```

健康守护定时器：

```text
openclaw-hermes-watchdog.timer
```

诊断命令：

```bash
openclaw-hermes-doctor check
openclaw-hermes-doctor smoke
```

说明：

- `check` 会检查桥梁服务、OpenClaw、Hermes 和关键环境变量。
- `smoke` 会额外分别调用 OpenClaw 和 Hermes 模型，确认两条模型链路都能回复。
- `watchdog` 由 systemd timer 每 5 分钟执行一次；如果桥梁服务 `/health` 不通，会自动重启 `openclaw-feishu-bridge`。

常用命令：

```bash
systemctl status openclaw-feishu-bridge --no-pager -l
systemctl restart openclaw-feishu-bridge
systemctl stop openclaw-feishu-bridge
systemctl start openclaw-feishu-bridge
journalctl -u openclaw-feishu-bridge -n 100 --no-pager
systemctl list-timers openclaw-hermes-watchdog.timer --no-pager
journalctl -u openclaw-hermes-watchdog -n 100 --no-pager
```

服务监听：

```text

Nginx 对外入口：0.0.0.0:80
```

## 6. 环境变量文件

位置：

```text
/etc/openclaw-feishu-bridge.env
```

里面包含：

```text
PORT=8788
GITHUB_OWNER=Inventionyin
GITHUB_REPO=OpenclawHomework
GITHUB_WORKFLOW_ID=ui-tests.yml
GITHUB_REF_NAME=main
UI_TEST_RUN_MODE=contracts
UI_TEST_TARGET_REPOSITORY=Inventionyin/UItest
UI_TEST_TARGET_REF=main
UI_TEST_BASE_URL=http://127.0.0.1:5173
OPENCLAW_PARSE_ENABLED=true
OPENCLAW_MODEL=xfyun/astron-code-latest
OPENCLAW_CHAT_ENABLED=true
HERMES_FALLBACK_ENABLED=true
HERMES_BIN=hermes
HERMES_PROVIDER=custom
HERMES_MODEL=astron-code-latest
FEISHU_RESULT_NOTIFY_ENABLED=true
FEISHU_APP_ID=...
FEISHU_APP_SECRET=...
FEISHU_CARD_ENABLED=true
FEISHU_WEBHOOK_ASYNC=true
FEISHU_ENV_FILE=/etc/openclaw-feishu-bridge.env
FEISHU_REQUIRE_BINDING=true
FEISHU_AUTOMATION_RECEIPT_ENABLED=false
FEISHU_DEDUP_ENABLED=true
FEISHU_DEDUP_TTL_MS=300000
GITHUB_TOKEN=...
```

修改后需要重启：

```bash
systemctl restart openclaw-feishu-bridge
```

说明：

- `OPENCLAW_PARSE_ENABLED=false` 时，只支持固定格式指令，例如 `/run-ui-test main contracts`。
- `OPENCLAW_PARSE_ENABLED=true` 时，会先尝试固定格式；固定格式不匹配时，再调用 OpenClaw 把自然语言解析成测试参数。
- `OPENCLAW_CHAT_ENABLED=true` 时，普通聊天消息会调用 OpenClaw 生成简短中文回复，不会触发测试。
- `HERMES_FALLBACK_ENABLED=true` 时，如果 OpenClaw 自然语言解析或普通聊天失败，会自动改用 Hermes。
- 当前云服务器已经安装 OpenClaw，并已验证自然语言解析可用。
- 当前云服务器已经安装 Hermes Agent，并配置到同一个讯飞 CodingPlan OpenAI-compatible endpoint。
- `FEISHU_RESULT_NOTIFY_ENABLED=true` 且配置飞书 App ID / App Secret 后，服务会轮询 GitHub Actions 并主动发送最终结果到飞书。
- `FEISHU_CARD_ENABLED=true` 时，结果通知使用飞书卡片。
- `FEISHU_WEBHOOK_ASYNC=true` 时，飞书回调会快速返回，后台继续跑测试，避免飞书请求超时。
- `FEISHU_ENV_FILE` 用于 `绑定我` 命令写入 `FEISHU_ALLOWED_USER_IDS`。
- `FEISHU_REQUIRE_BINDING=true` 时，没有绑定用户前不能触发测试；在飞书发送 `绑定我` 后才允许当前用户触发。
- `FEISHU_AUTOMATION_RECEIPT_ENABLED=false` 时，关闭自动化指令的“已收到”即时回复，只保留最终结果卡片，聊天体验更接近一问一答。
- `FEISHU_DEDUP_ENABLED=true` 会忽略短时间内重复投递的同一条飞书事件，避免同一条消息触发两次回复或两次测试。
