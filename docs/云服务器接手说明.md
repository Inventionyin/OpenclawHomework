# 云服务器接手说明

## 1. 服务器用途

这台服务器用于部署 OpenClawHomework 的飞书 Webhook 桥梁服务。

当前链路：

```text
飞书 / 模拟 Webhook
  -> http://38.76.178.91/webhook/feishu
  -> Nginx
  -> Node 桥梁服务 127.0.0.1:8788
  -> GitHub API workflow_dispatch
  -> GitHub Actions 执行 UItest
```

## 2. 服务器基本信息

```text
系统：Debian GNU/Linux 13
公网 IP：38.76.178.91
SSH 用户：root
SSH 端口：22
```

注意：

- 不要把服务器密码、GitHub Token、飞书 App Secret 写进 GitHub 仓库。
- GitHub Token 当前存放在服务器 `/etc/openclaw-feishu-bridge.env`，文件权限为 `600`。

## 5. 服务信息

systemd 服务名：

```text
openclaw-feishu-bridge
```

常用命令：

```bash
systemctl status openclaw-feishu-bridge --no-pager -l
systemctl restart openclaw-feishu-bridge
systemctl stop openclaw-feishu-bridge
systemctl start openclaw-feishu-bridge
journalctl -u openclaw-feishu-bridge -n 100 --no-pager
```

服务监听：

```text
Node 桥梁服务：127.0.0.1:8788
Nginx 对外入口：0.0.0.0:80
```

为什么不是 8787：

```text
服务器上已有一个 Python 服务占用了 127.0.0.1:8787。
因此 OpenClawHomework 桥梁服务改用 8788。
```

## 6. 环境变量文件

位置：

```text
/etc/openclaw-feishu-bridge.env
```

里面包含：

```text
PORT=8788
GITHUB_OWNER=Inventionyin
GITHUB_REPO=OpenclawHomework
GITHUB_WORKFLOW_ID=ui-tests.yml
GITHUB_REF_NAME=main
UI_TEST_RUN_MODE=contracts
UI_TEST_TARGET_REPOSITORY=Inventionyin/UItest
UI_TEST_TARGET_REF=main
UI_TEST_BASE_URL=http://127.0.0.1:5173
GITHUB_TOKEN=...
```

修改后需要重启：

```bash
systemctl restart openclaw-feishu-bridge
```

## 9. 模拟飞书请求

PowerShell 示例：

```powershell
$payload = @{
  event = @{
    sender = @{
      sender_id = @{
        open_id = "server-public-test"
      }
    }
    message = @{
      content = '{"text":"/run-ui-test main contracts"}'
    }
  }
} | ConvertTo-Json -Depth 10

Invoke-RestMethod `
  -Method Post `
  -Uri "http://38.76.178.91/webhook/feishu" `
  -ContentType "application/json" `
  -Body $payload
```

成功返回：

```json
{
  "ok": true,
  "message": "UI 自动化测试已触发：分支 main，模式 contracts",
  "actionsUrl": "https://github.com/Inventionyin/OpenclawHomework/actions/workflows/ui-tests.yml",
  "targetRepository": "Inventionyin/UItest",
  "targetRef": "main",
  "runMode": "contracts"
}
```

## 11. 下一步：绑定域名和 HTTPS

飞书正式事件订阅通常需要 HTTPS。

建议使用子域名：

```text
openclaw.你的域名.com
```

DNS 配置：

```text
类型：A
主机记录：openclaw
记录值：38.76.178.91
```

等 DNS 生效后，在服务器安装证书：

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d openclaw.你的域名.com
```

飞书 Webhook 地址改成：

```text
https://openclaw.你的域名.com/webhook/feishu
```
