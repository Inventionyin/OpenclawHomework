# 飞书桥梁服务使用说明

## 0. 本轮大神版核心升级（飞书侧）

本轮在飞书交互侧落地三项核心升级：

- 自然语言总控：`规则强保护 + 模型二判 + 规则兜底`。
- 任务中枢：`今日任务 / 历史任务 / 失败复盘 / 下一步计划`。
- 文员日报闭环：`今日总结 / 明日计划 / 失败诊断 / token 消耗 / 邮件归档`。

可复制飞书口令：

```text
文员，查看今天任务中枢
文员，给我最近一次失败复盘和下一步计划
文员，生成今日总结并附上 token 消耗
文员，查看今天邮箱工作台
文员，列出待审批邮件
文员，生成 ClawEmail 每日报告
文员，发送今天日报到邮箱
```

## 1. 作用

桥梁服务负责把飞书群里的指令转换成 GitHub Actions 调度请求。

### 1.1 Dify 测试助手集成（Task C）

职责分工：

```text
Hermes   = 协调者（coordinator），负责理解需求、拆解任务、组织回复
Dify     = 测试知识与生成层，负责测试方案/用例/复盘建议生成
OpenClaw = 执行层（executor），负责触发脚本、调度 CI、回传结果
```

建议环境变量（仅变量名，不在文档写真实值）：

```text
DIFY_TESTING_ASSISTANT_ENABLED=true
DIFY_TESTING_ASSISTANT_URL=http://127.0.0.1
DIFY_TESTING_ASSISTANT_API_KEY=app-xxxx
DIFY_TESTING_ASSISTANT_TIMEOUT_MS=30000
DIFY_TESTING_ASSISTANT_RESPONSE_MODE=blocking
HERMES_COORDINATOR_ENABLED=true
OPENCLAW_EXECUTION_ENABLED=true
```

`DIFY_TESTING_ASSISTANT_URL` 可以填 Dify 服务 base URL（如 `http://127.0.0.1`），也可以填完整接口 `http://127.0.0.1/v1/chat-messages`；程序会自动归一化。

飞书示例话术：

```text
文员，先让 Dify 生成这次回归测试清单，再由 OpenClaw 执行 smoke。
文员，用 Dify 总结这次失败原因，并给 OpenClaw 一个最小复现步骤。
文员，把这个需求拆成：Dify 出测试点，OpenClaw 跑验证。
```

localhost 注意事项：

```text
飞书回调不能直接访问你本机 localhost/127.0.0.1。
本地联调需使用内网穿透或部署到可公网 HTTPS 访问地址。
```

安全要求：

```text
文档和聊天总结中禁止写入 API Key、Token、App Secret、密码。
敏感值只放服务器环境文件或安全密钥管理系统。
```

当前链路：

```text
飞书消息
  -> POST /webhook/feishu
  -> 解析 /run-ui-test main contracts
  -> 可选：调用 OpenClaw 解析自然语言
  -> 调用 GitHub workflow_dispatch
  -> 触发 UI Automation Dispatch
  -> GitHub Actions 拉取电商前端和 UItest
  -> GitHub Actions 启动电商前端并生成测试报告
  -> 可选：轮询 GitHub Actions 结果并主动发回飞书
```

## 2. 当前已实现能力

- 支持飞书 challenge 校验响应
- 支持解析飞书事件中的 `event.message.content`
- 支持简单指令：

```text
/run-ui-test main contracts
```

- 支持三个运行模式：
  - `contracts`
  - `smoke`
  - `all`
- 支持用户 allowlist 授权
- 支持调用 GitHub API 触发 Actions
- 支持可选 OpenClaw 自然语言解析
- 支持查询 GitHub Actions run 链接
- 支持可选飞书主动结果通知
- 支持飞书 interactive card 结果卡片
- 支持普通聊天：`你好`、`帮助`、自由问答
- 支持 `绑定我`，把测试触发权限限制到当前飞书用户
- 支持 Agent Router，把消息分流到聊天、UI 测试、运维、文档、记忆等轻量 agent
- 支持安全记忆文件和技能文档，帮助新 AI 接手项目
- 支持 `/status`、`/health`、`/watchdog`、`/logs` 等白名单运维查询
- 支持文字生图和图片编辑：可以直接生图，也可以发旧照片让机器人修复
- 支持文件通道登记：外部 Bridge 可以把报告、截图、trace 保存到安全根目录，再把路径交给文员 agent
- 支持 ClawEmail 邮箱工作台：读取最近收件快照、结合发信账本、分类邮件、列出待审批邮件、生成 ClawEmail 风格日报预览
- 支持 RD-Agent-lite 研发循环、白名单网页正文抽取、skflow-lite 技能流程，把开源项目思路内化到任务中枢
- 预留微信 Web Bridge 独立入口，详见 `docs/微信WebBridge接入说明.md`

自然语言示例：

```text
帮我跑一下 main 分支的 UI 自动化冒烟测试
```

启用 OpenClaw 解析后，会转换成：

```json
{"targetRef":"main","runMode":"smoke"}
```

## 3. 本地启动方式

在项目根目录执行：

```powershell
$env:GITHUB_TOKEN = (Get-Content -Raw "C:\Users\殷文炫\Desktop\Github_TOKEN.txt").Trim()
npm run bridge:feishu
```

如果要在本地启用 OpenClaw 自然语言解析：

```powershell
$env:OPENCLAW_PARSE_ENABLED = "true"
$env:OPENCLAW_MODEL = "xfyun/astron-code-latest"
npm run bridge:feishu
```

默认监听：

```text
http://127.0.0.1:8787
```

健康检查：

```text
GET http://127.0.0.1:8787/health
```

飞书 Webhook 地址：

```text
POST http://127.0.0.1:8787/webhook/feishu
```

如果要让飞书公网访问本地服务，需要再使用内网穿透工具，把本地 `8787` 暴露成公网 HTTPS 地址。

当前已经部署到云服务器：

```text
https://openclaw.evanshine.me/health
https://openclaw.evanshine.me/webhook/feishu
https://openclaw.evanshine.me/webhook/feishu/openclaw
https://openclaw.evanshine.me/webhook/feishu/hermes
https://hermes.evanshine.me/webhook/feishu
```

服务器上实际 Node 服务端口是 `8788`，因为 `8787` 已被原有 Python 服务占用。外部访问不需要带端口，统一走 Nginx 的 HTTPS `443` 端口。

路由说明：

- `/webhook/feishu`：兼容旧 OpenClaw 机器人，默认 OpenClaw。
- `/webhook/feishu/openclaw`：OpenClaw 专用机器人入口，默认 OpenClaw，Hermes 作为 fallback。
- `/webhook/feishu/hermes`：Hermes 专用机器人入口，默认 Hermes，OpenClaw 作为 fallback。
- `https://hermes.evanshine.me/webhook/feishu`：Hermes 独立域名入口，等价于上面的 Hermes 专用入口，更推荐填到 Hermes 机器人后台。

## 5. 授权配置

如果不配置 `FEISHU_ALLOWED_USER_IDS`，本地开发默认允许所有用户触发。

如果要限制用户：

```powershell
$env:FEISHU_ALLOWED_USER_IDS = "open_id_1,open_id_2"
```

只有 allowlist 里的飞书用户 ID 才能触发 UI 自动化。

如果还不知道自己的飞书 `open_id`，推荐先打开：

```powershell
$env:FEISHU_REQUIRE_BINDING = "true"
```

这样在没有绑定用户前，任何测试触发请求都会被拒绝；你在飞书里发送 `绑定我` 后，服务会把当前发送人的 `open_id` 写入 allowlist。

## 7. Agent Router、记忆和技能

桥梁服务支持轻量 Agent Router。它不是多个并发 OpenClaw CLI，而是在同一个 Node 服务里按意图分工，避免再次出现多个模型请求抢同一个 session 的问题。

当前 agent 分工：

```text
chat-agent    普通聊天，不默认携带完整项目记忆
ui-test-agent UI 自动化触发，继续走原有 GitHub Actions 流程
ops-agent     安全状态检查，只允许白名单动作
doc-agent     回答任务进度、交接和文档问题
memory-agent  读取或记录非敏感记忆
```

危险操作有置信度保护：明确的“重启你自己”“修复 Hermes”会执行；像“你重起一下”“帮我搞一下”这类模糊说法会先要求确认，不会直接重启或修复。

自然语言总控判定顺序（飞书口径）：

1. 规则强保护：先判断是否涉及高风险动作、越权动作、否定/教程语境。
2. 模型二判：对模糊句做意图复判，只输出建议或确认，不直接落危险执行。
3. 规则兜底：模型不确定时，回退安全规则，默认拒绝或降级成只读说明。

可复制飞书口令：

```text

请问怎么重启你自己
```

硬盘清理是两步流程：先说“看看哪些东西占硬盘”或“khoj 可以清理吗”，机器人只做盘点并保存本轮候选项；再说“确认清理第 1 个”或“清理 khoj”，才会执行白名单内的清理动作。当前白名单包括 Khoj 目录、npm 缓存、旧日志和临时文件，不支持从自然语言执行任意删除命令。

飞书回复展示增强：


开启后，桥接服务会在文本回复底部追加状态和耗时，并在服务日志里记录收到消息、路由、模型调用、发送飞书等阶段耗时。OpenClaw 官方 Feishu channel 也可以同时开启：

```bash
openclaw config set channels.feishu.streaming true
openclaw config set channels.feishu.footer.elapsed true
openclaw config set channels.feishu.footer.status true
```

```text

STREAMING_MODEL_API_KEY=...

```

如果前面加了 LiteLLM 或其它 OpenAI-compatible 模型网关，也可以把网关作为统一入口：

```text

LITELLM_BASE_URL=http://127.0.0.1:4000/v1

LITELLM_SIMPLE_MODEL=longcat/lite
LITELLM_THINKING_MODEL=claude/sonnet
STREAMING_MODEL_ENDPOINT_MODE=chat_completions
```

等价通用变量也支持：`MODEL_GATEWAY_BASE_URL`、`MODEL_GATEWAY_API_KEYS`、`MODEL_GATEWAY_MODEL`、`MODEL_GATEWAY_SIMPLE_MODEL`、`MODEL_GATEWAY_THINKING_MODEL`。

优先级：`STREAMING_MODEL_*` > `MODEL_GATEWAY_* / LITELLM_*` > `OPENAI_* / XFYUN_*`。这样可以先让 Hermes 走网关，OpenClaw 保持原链路，后面再逐步迁移。

说明：

- 只对普通聊天启用流式；`/status`、UI 自动化、修复服务器等命令仍走稳定的确定性回复。
- 桥梁会优先尝试 `/responses`，如果接口不是 SSE 或不兼容，会自动降级到 `/chat/completions`。
- 讯飞 CodingPlan 当前实测可用的是 `/chat/completions` 的 `text/event-stream`，`/responses` 会返回 One API HTML 页面，不适合作为流式接口。
- 飞书侧先发一条“正在思考...”占位消息，再用 `PATCH /im/v1/messages/{message_id}` 更新同一条消息，避免刷屏。
- 流式失败时会自动回退到原来的 OpenClaw/Hermes CLI 一次性回复。
- 如果模型流式接口返回 usage，桥梁会把每次普通聊天的模型、tier、endpoint、耗时、输入/输出/总 token 写入 JSONL 账本。账本不记录 API key、App Secret、GitHub token、邮箱授权码。
- 邮件动作也有独立账本。开启 `MAIL_LEDGER_ENABLED=true` 后，日报、UI 自动化报告、archive/eval/report 等邮箱动作每次真实发信都会记录机器人名称、动作、SMTP 通道、收件人、归档邮箱、主题和发送状态；不会保存 SMTP 授权码、API key、token 或完整错误栈。

查看最近 10 条账本：

```bash
tail -n 10 /var/log/openclaw-homework/usage-ledger.jsonl
tail -n 10 /var/log/openclaw-homework/mail-ledger.jsonl
```

按机器人和模型汇总 token：

```bash
node -e "const fs=require('fs');const f='/var/log/openclaw-homework/usage-ledger.jsonl';const m={};for(const l of fs.readFileSync(f,'utf8').trim().split(/\n/)){const x=JSON.parse(l);const k=[x.assistant,x.model].join(' / ');m[k]??={calls:0,tokens:0,ms:0};m[k].calls++;m[k].tokens+=x.totalTokens||0;m[k].ms+=x.modelElapsedMs||0;}console.table(m)"
```


改完后重启桥梁服务即可生效。

注意：上面这组 `MAILBOX_ACTION_*_TO` 只负责“收件地址覆盖”。如果你要调整 `report / daily` 使用哪个发件 SMTP，请改 `REPORT_SMTP_*` 和 `MAIL_ACTION_PROVIDER_OVERRIDES`，不要把这两层混在一起。

如果使用 Hermes 独立飞书机器人，还需要在服务器环境变量里配置：

```text
HERMES_FEISHU_APP_ID=...
HERMES_FEISHU_APP_SECRET=...
```

这样 `https://hermes.evanshine.me/webhook/feishu` 收到的消息会用 Hermes 机器人的身份回复，而不是 OpenClaw 机器人的身份。

## 9. Watchdog 守护巡检

仓库提供了 `scripts/server-watchdog.js` 和 `scripts/install-watchdog.sh`。

用途：

- 定时检查桥梁服务 `/health`。
- 健康检查失败时自动 `systemctl restart` 对应服务。
- 扫描 Nginx access log 中的飞书 webhook POST。
- 发现非 `200` 回调或请求风暴时记录原因，避免再次出现夜间消息轰炸时没人发现。

本地手动运行：

```powershell
npm run watchdog -- --service openclaw-feishu-bridge --health-url http://127.0.0.1:8788/health
```

服务器安装定时器：

```bash
bash scripts/install-watchdog.sh \
  --unit-name openclaw-homework-watchdog \
  --bridge-service openclaw-feishu-bridge \
  --env-file /etc/openclaw-feishu-bridge.env
```

Hermes 服务器把 `unit-name`、`bridge-service`、`env-file` 换成 Hermes 对应值。

## 9.1 30 分钟热点/福利雷达

`scripts/hot-monitor.js` 用于短周期扫描热点和福利活动。它和每日流水线不同：每日流水线负责每天汇总一次；热点雷达负责每 30 分钟比较快照，只在发现新增高价值项目或福利活动时主动飞书提醒。

监听范围：

```text
GitHub / Hacker News / RSS / Tavily / Brave / SerpApi 技术热点
AI Agent、UI 自动化、软件测试、电商、邮箱平台相关项目
免费 token、模型额度、云服务器/GPU 额度、免费试用、会员体验、内测邀请
Product Hunt、Hugging Face、GitHub Blog、Cloudflare Blog、Linux.do/V2EX/贴吧等可搜索来源
英文标题会补中文理解和中文摘要，方便直接判断是否值得点开
```

搜索 API 可选配置：

```text
HOT_MONITOR_SEARCH_ENABLED=true
HOT_MONITOR_SEARCH_QUERIES=free LLM API credits,AI credits free trial,GPU credits for developers,cloud server credits startup,site:linux.do token 免费 额度,site:v2ex.com 免费 token 服务器,site:tieba.baidu.com AI 免费 token
HOT_MONITOR_SEARCH_PER_QUERY=3
HOT_MONITOR_SEARCH_MAX_QUERIES=8
HOT_MONITOR_TAVILY_API_KEY=...
HOT_MONITOR_BRAVE_API_KEY=...
HOT_MONITOR_SERPAPI_API_KEY=...
HOT_MONITOR_SERPAPI_ENGINE=baidu
HOT_MONITOR_SEARCH_LANG=zh-hans
HOT_MONITOR_SEARCH_COUNTRY=HK
```

真实 key 只放服务器 env 文件，不写进仓库。SerpApi 可用于百度/贴吧类搜索；Tavily 更适合 Agent 摘要；Brave 更适合最近 24 小时/7 天的全网热点。

可选开源搜索聚合：

```text
HOT_MONITOR_SEARXNG_URL=https://你的-searxng/search-入口域名
HOT_MONITOR_SEARXNG_ENABLED=true
```

SearXNG 不是自带全网索引，它是元搜索聚合器；适合把多个搜索源统一成一个入口。真正长期有价值的是把搜索候选、浏览器验证结果和接口抓包沉淀到本地协议资产库。

热点候选入库和电商玩法：

```text
搜索 API / SearXNG 找线索
Hermes hot-monitor 去重、中文化、过滤过期活动
browser-agent 打开自有平台或公开候选页做验证
protocol-asset-store 保存候选链接和网络请求
buildProtocolTestCases 把协议资产转成接口契约测试
OpenClaw 触发 UItest GitHub Actions 跑电商 UI 自动化
Hermes 文员 Agent 生成客服训练数据、日报和复盘邮件
```

常用自然语言：

```text
帮我搜今天电商测试和 AI 客服相关福利，过期的不要提醒
把最新福利候选做浏览器验证并协议入库
打开自有电商平台登录页做浏览器验证和抓包
把最近协议资产转成接口契约测试用例
跑 main 分支 UI 自动化并把 Allure 报告发邮箱
基于今天失败样本生成一批电商客服训练数据
```

本地生成电商 Agent 玩法清单：

```bash
npm run qa:assets -- data/qa-assets
```

协议资产最小验收：

```bash
HOT_MONITOR_FEISHU_NOTIFY_ENABLED=false PROTOCOL_ASSET_DIR=/var/lib/openclaw-homework/protocol-assets node scripts/hot-monitor.js --force --env-file /etc/hermes-feishu-bridge.env
ls -lah /var/lib/openclaw-homework/protocol-assets
node -e "const { buildProtocolAssetReport, buildProtocolTestCases } = require('./scripts/protocol-asset-store'); console.log(JSON.stringify(buildProtocolAssetReport({ tag: 'hot-monitor' }, { dir: '/var/lib/openclaw-homework/protocol-assets' }), null, 2)); console.log(JSON.stringify(buildProtocolTestCases({ tag: 'hot-monitor' }, { dir: '/var/lib/openclaw-homework/protocol-assets' }), null, 2));"
```

注意：`--dry-run` 只验证雷达流程，不会真正写入新告警冷却时间和协议资产；真实定时运行命中告警后才会自动入库。上面的验收命令临时关闭飞书通知，只验证入库。

安装示例：

```bash
bash scripts/install-hot-monitor.sh \
  --unit-name hermes-hot-monitor \
  --env-file /etc/hermes-feishu-bridge.env \
  --state-file /var/lib/openclaw-homework/hot-monitor-state.json \
  --output-file /var/lib/openclaw-homework/hot-monitor-latest.json \
  --on-calendar '*:0/30'
```

默认不发送空报告，避免每 30 分钟刷屏。需要调试时：

```bash
node scripts/hot-monitor.js --dry-run --force --env-file /etc/hermes-feishu-bridge.env
systemctl list-timers '*hot-monitor*' --no-pager
journalctl -u hermes-hot-monitor --since '2 hours ago' --no-pager
grep -E 'HOT_MONITOR_SEARCH|TAVILY_API_KEY|BRAVE_SEARCH_API_KEY|SERPAPI_API_KEY|HOT_MONITOR_TAVILY|HOT_MONITOR_BRAVE|HOT_MONITOR_SERPAPI|SEARXNG|PROTOCOL_ASSET_DIR' /etc/hermes-feishu-bridge.env | sed -E 's/=.*/=***/'
```
