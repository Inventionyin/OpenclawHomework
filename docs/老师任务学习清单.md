老师布置的任务可以理解为一句话：

> 通过飞书给 OpenClaw 发自然语言指令，让 OpenClaw 去触发 GitHub Actions，然后在 GitHub 上执行 UI 自动化测试，最后把结果反馈回飞书。


## 5. 第三阶段：配置 GitHub Actions

这一阶段的目标是让 GitHub 自己执行测试。

### 任务 4：创建 workflow 文件

要创建文件：

```text
.github/workflows/ui-tests.yml
```

推荐内容：

```yaml
name: UI Automation Tests

on:
  workflow_dispatch:
    inputs:
      test_env:
        description: "测试环境"
        required: true
        default: "staging"

jobs:
  ui-tests:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Install Playwright browsers
        run: npx playwright install --with-deps

      - name: Run UI tests
        run: npx playwright test
        env:
          TEST_ENV: ${{ github.event.inputs.test_env }}

      - name: Upload Playwright report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/
```

你需要学会：

- GitHub Actions 是什么
- `.github/workflows` 的作用
- `workflow_dispatch` 为什么可以被手动/API 触发
- `jobs`、`steps`、`uses`、`run` 分别是什么意思

验收标准：

- GitHub Actions 页面出现 `UI Automation Tests`
- 可以手动点击 `Run workflow`
- 执行后可以看到成功或失败日志

## 6. 第四阶段：通过 GitHub API 触发 Actions

这一阶段的目标是不用手点 GitHub 页面，而是用代码触发 workflow。

### 任务 5：创建 GitHub Token

要做的事情：

创建 GitHub Fine-grained personal access token。

建议权限：

- Repository access：只选择这个作业仓库
- Actions：Read and write
- Contents：Read
- Metadata：Read

注意：

- Token 不能提交到 GitHub
- Token 不能写进代码
- Token 只放在环境变量或 OpenClaw 配置里

你需要学会：

- Token 是什么
- 为什么不能把 Token 写进代码
- 最小权限原则是什么

验收标准：

- 拿到一个 GitHub Token
- Token 保存到本地环境变量或 OpenClaw 配置

### 任务 6：编写触发脚本

建议创建文件：

```text
scripts/trigger-ui-tests.js
```

脚本目标：

- 读取 GitHub Token
- 调用 GitHub API
- 触发 `ui-tests.yml`
- 传入测试环境参数

核心 API 是：

```text
POST /repos/{owner}/{repo}/actions/workflows/{workflow_id}/dispatches
```

你需要学会：

- 什么是 REST API
- 什么是 GitHub Actions workflow dispatch
- 环境变量怎么传给 Node.js

验收标准：

- 执行脚本后，GitHub Actions 页面出现新任务
- 不需要手动点击 `Run workflow`
