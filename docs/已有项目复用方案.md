# 已有电商项目与 UI 自动化仓库复用方案

## 1. 已有仓库

| 仓库 | 地址 | 用途 |
|---|---|---|
| UI 自动化测试脚本 | https://github.com/Inventionyin/UItest | Playwright + TypeScript + Allure 测试工程 |
| AI 智能客服电商平台 | https://github.com/dengzhekun/projectku-web | Vue 3 前端、Spring Boot 后端、FastAPI AI 客服服务 |
| 当前作业仓库 | https://github.com/Inventionyin/OpenclawHomework | OpenClaw / 飞书 / GitHub Actions 调度作业入口 |

## 2. 推荐仓库分工

不要把三个项目强行合成一个大仓库。

推荐分工如下：

```text
OpenclawHomework
  负责：作业说明、飞书指令流程、GitHub Actions 调度、触发脚本

UItest
  负责：Playwright UI 自动化脚本、Allure 报告、测试分层结构

projectku-web
  负责：被测电商平台、AI 智能客服、真实业务页面和接口
```

这样更接近真实团队协作方式：

- 应用项目和测试项目可以独立维护
- 调度仓库只负责远程触发和结果汇总
- 老师能清楚看到“指令触发 GitHub Actions 执行 UI 自动化”的链路

## 3. 复用评估

### 可以直接复用

来自 `UItest`：

- Playwright + TypeScript 测试框架
- `pages/` Page Object 分层
- `fixtures/` 公共测试夹具
- `data/` 测试数据模板
- `helpers/` 等待、断言、报告工具
- Allure 报告生成脚本
- Playwright HTML 报告
- GitHub Actions 基线检查思路

来自 `projectku-web`：

- Vue 3 电商前端
- 已有本地 smoke 测试
- 默认测试账号：`user@example.com` / `123456`
- 默认前端地址：`http://127.0.0.1:5173`
- 默认后端 API：`http://127.0.0.1:8080/api`
- AI 客服相关页面、接口和文档

### 需要改造后复用

- `UItest` 中依赖真实页面的测试，需要确认和 `projectku-web` 当前页面 locator 是否完全一致
- 登录、购物车、结算、订单等流程，需要保证测试数据稳定
- 如果在 GitHub Actions 中跑真实 UI 测试，需要先启动 `projectku-web` 的前端、后端和 MySQL
- 如果要测试 AI 客服，需要额外启动 `ai-service`，并配置模型或 mock 服务

### 不建议直接复制

- 不要直接把 `UItest` 整个仓库复制进 `OpenclawHomework`
- 不要把 `projectku-web` 整个仓库复制进 `OpenclawHomework`
- 不要把真实 `.env`、Token、模型 Key、数据库密码提交到作业仓库
- 不要一开始就在 GitHub Actions 里跑完整 AI 客服链路，太容易被环境和模型服务卡住

## 4. 当前已经采用的方案

当前作业仓库已经新增：

```text
.github/workflows/ui-tests.yml
scripts/trigger-ui-tests.js
tests/trigger-ui-tests.test.js
```

作用：

- `ui-tests.yml`：允许手动或 API 触发 UI 自动化
- workflow 会拉取 `Inventionyin/UItest`
- 默认运行 `tests/contracts`
- 生成并上传 Playwright / Allure 报告
- `trigger-ui-tests.js`：用于 OpenClaw 或本地命令触发 GitHub Actions

第一版为什么只跑 `contracts`：

- 不依赖真实电商平台启动
- 不依赖测试账号
- 不依赖数据库
- 可以先证明 GitHub Actions 调度和报告生成链路是通的

后续再扩展到：

- `smoke`：跑登录、导航等轻量真实 UI 测试
- `all`：跑完整 UI 自动化测试集

## 5. 适合给老师解释的话

本作业没有把已有项目代码全部复制到一个仓库，而是采用三仓库协作方式：

```text
OpenclawHomework 作为调度仓库
UItest 作为 UI 自动化测试仓库
projectku-web 作为被测电商平台仓库
```

通过 `workflow_dispatch`，OpenClaw 或飞书桥梁服务只需要触发 `OpenclawHomework` 的 GitHub Actions，就能远程拉取 UI 自动化测试仓库并执行测试。这样更符合实际工程中的自动化测试平台设计。

## 6. 下一步

下一步推荐：

1. 先把当前作业仓库推送到 GitHub
2. 在 GitHub Actions 页面手动运行 `UI Automation Dispatch`
3. 确认能生成 `allure-report` artifact
4. 创建 GitHub Token
5. 本地运行 `npm run trigger:ui` 触发 workflow
6. 再接入 OpenClaw / 飞书机器人
